检视XXO
看正在跑的XXO -> 按某条件排序 -> 检视问题
检视要点:CPU 占用过大; 漏配置了某台机器; CPU,MEM,VSZ或RSS占用过大; 很久没有重启; 发现废弃部署的xxo等;
看XXO cont_config 配置 -> 按某条件排序 -> 检视问题
检视要点: 请求/返回/错误ITIL没有配置; pthsize 过大; RestartItil,AuthcCheck没有配置; 配置文件更新日期很古老; session数和超时值设置不合理; 没有配置新的统一log等.
看XXO ServiceConfig 配置 -> 按某条件排序 -> 检视问题
检视要点: 设置的Size值不合理, 统一取其msgq与堵塞监控比较检查, 废弃的远程xxo的配置要注意及时清理(msgq为0).
看XXO container 版本 -> 按某条件排序 -> 检视问题
检视要点: container 版本号过老或者没有按规范用link来部署
看XXO log 的关闭状态 -> 按某条件排序 -> 检视问题
检视要点:所有业务进程要关闭debug log, 繁忙的业务要关闭run log, 最繁忙业务的perform log 也要关闭
看正在跑的XXO -> 操作检视某个xxo -> 检视问题
检视要点:整个xxo全方位的体检

检视机器
看全部机器上的SHM状态 -> 按某条件排序 -> 检视问题
检视要点:key 为空的死SHM, attach 数为0的废弃shm等
看全部机器上的Msgq状态 -> 按某条件排序 -> 检视问题
检视要点:key 为空的死msgq, 堵塞数过大的废弃msgq等
看全部机器上的TOP状态 -> 按某条件排序 -> 检视问题
检视要点:mem剩余过少,swap 剩余过少, 机器很久没有重启, 僵死的进程有很多, 同时并发的任务数有很多.
看全部机器上的硬盘状态 -> 按某条件排序 -> 检视问题
检视要点:硬盘分区空间剩余过少
